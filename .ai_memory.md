- **Dominio:** (pendiente de configurar)

## Reglas de Oro
- El sitio debe ser rápido, accesible y con buen SEO desde el día uno.
- Usar Server Components de Next.js por defecto; Client Components solo cuando sea necesario (interactividad, hooks de estado).
- Los artículos del blog se escriben en MDX (Markdown + JSX) para poder incluir componentes interactivos dentro de los posts.
- No instalar librerías sin consultarme primero.
- Mantener el JavaScript vanilla — no migrar a TypeScript a menos que se decida explícitamente.
- Diseño mobile-first. Todo se prueba en viewport de 375px antes de escalar.

## Decisiones de Arquitectura
- [2026-04-02] **Framework:** Se eligió Next.js sobre Vite por la necesidad de SSG/SSR para SEO del blog y portfolio. Los componentes React de la landing anterior en Vite se reutilizan.
- [2026-04-02] **Estilos:** Tailwind CSS por velocidad de desarrollo y consistencia. No se usa CSS modules ni styled-components.
- [2026-04-02] **Blog engine:** MDX con contentlayer o next-mdx-remote para renderizar artículos desde archivos locales (sin CMS externo por ahora).
- [2026-04-02] **Estructura de rutas:** App Router de Next.js (`/app`). Cada sección es un route group.

## Estructura de Rutas
